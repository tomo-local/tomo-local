## Commands

### Generate resume (README.md)

Build the `resume/README.md` from parts using the Go build tool:

```sh
pnpm generate
# or directly:
./resume/src/build/doc -config ./resume/build.config.json -output ./resume/README.md
```

The binary must be compiled first if it doesn't exist:

```sh
cd resume/src && go build -o build/doc .
```

### Lint

```sh
pnpm lint
```

This runs two linters in sequence:
- `textlint` вҖ” checks Japanese writing style (grammar, punctuation, spacing rules)
- `markdownlint` вҖ” checks Markdown formatting

### Run tests (Go)

```sh
cd resume/src && go test ./...
```

## Architecture

This repo has two concerns:

1. **GitHub profile README** (`README.md` at root) вҖ” a static Markdown file with badges and GitHub stats widgets. Edited directly.

2. **иҒ·еӢҷзөҢжӯҙжӣё (resume)** (`resume/README.md`) вҖ” auto-generated by the Go build tool from parts files. **Do not edit `resume/README.md` directly**; edit the source parts instead.

### Resume build system

```
resume/
в”ңв”Җв”Җ build.config.json       # Defines section order and which part files to include
в”ңв”Җв”Җ src/
в”Ӯ   в”ңв”Җв”Җ build.go            # Go CLI: reads config, concatenates parts вҶ’ README.md
в”Ӯ   в””в”Җв”Җ build_test.go       # Unit tests for loadConfig and build functions
в””в”Җв”Җ parts/
    в”ңв”Җв”Җ summary.md          # иҒ·еӢҷиҰҒзҙ„
    в”ңв”Җв”Җ skills.md           # гӮ№гӮӯгғ«гғ¬гғҷгғ«иЎЁ
    в”ңв”Җв”Җ career_table.md     # иҒ·еӢҷзөҢжӯҙдёҖиҰ§гғҶгғјгғ–гғ«
    в”ңв”Җв”Җ self_pr.md          # иҮӘе·ұPR
    в”ңв”Җв”Җ other_activities.md # гҒқгҒ®д»–гҒ®жҙ»еӢ•
    в””в”Җв”Җ companies/          # еҗ„зӨҫгҒ®и©ізҙ°зөҢжӯҙ (1гғ•гӮЎгӮӨгғ« per дјҡзӨҫ)
        в”ңв”Җв”Җ hireroo.md
        в”ңв”Җв”Җ shearwater.md
        в”ңв”Җв”Җ byard.md
        в””в”Җв”Җ strategit.md
```

The build tool reads `build.config.json`, resolves `file` / `files` paths relative to the config file's directory, and concatenates them with headings into `resume/README.md`.

### Adding or updating content

- To edit a section, modify the corresponding file under `resume/parts/`.
- To add a new company, create `resume/parts/companies/<name>.md` and add it to the `files` array in `build.config.json`.
- After editing, regenerate: `pnpm generate`.
- Then lint: `pnpm lint`.
