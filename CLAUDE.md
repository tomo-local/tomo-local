# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

### Generate resume (README.md)

Build the `resume/README.md` from parts using the Go build tool:

```sh
pnpm generate
# or directly:
./resume/src/build/doc -config ./resume/build.config.json -output ./resume/README.md
```

The binary must be compiled first if it doesn’t exist:

```sh
cd resume/src && go build -o build/doc .
```

### Lint

```sh
pnpm lint
```

This runs two linters in sequence:

- `textlint` — checks Japanese writing style (grammar, punctuation, spacing rules)
- `markdownlint` — checks Markdown formatting

### Run tests (Go)

```sh
cd resume/src && go test ./...
```

## Architecture

This repository has two concerns:

1. **GitHub profile README** (`README.md` at root) — a static Markdown file with badges and GitHub stats widgets. Edited directly

2. **職務経歴書 (resume)** (`resume/README.md`) — auto-generated by the Go build tool from parts files. **Do not edit `resume/README.md` directly**; edit the source parts instead

### Resume build tool

```
resume/
├── build.config.json       # Defines section order and which part files to include
├── src/
│   ├── build.go            # Go CLI: reads config, concatenates parts → README.md
│   └── build_test.go       # Unit tests for loadConfig and build functions
└── parts/
    ├── summary.md          # 職務要約
    ├── skills.md           # スキルレベル表
    ├── career_table.md     # 職務経歴一覧テーブル
    ├── self_pr.md          # 自己PR
    ├── other_activities.md # その他の活動
    └── companies/          # 各社の詳細経歴 (1ファイル per 会社)
        ├── hireroo.md
        ├── shearwater.md
        ├── byard.md
        └── strategit.md
```

The build tool reads `build.config.json`, resolves `file` / `files` paths relative to the config file’s directory, and concatenates them with headings into `resume/README.md`.

### Adding or updating content

- To edit a section, modify the corresponding file under `resume/parts/`
- To add a new company, create `resume/parts/companies/<name>.md` and add it to the `files` array in `build.config.json`
- After editing, regenerate: `pnpm generate`
- Then lint: `pnpm lint`
