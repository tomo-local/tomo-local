### 株式会社 BYARD (2023/01 ~ 2023/08)

業務を可視化し、カイゼンを容易にする業務設計プラットフォームの開発メンバーとして参画しました。最初は、フロントエンドの開発を担当し、途中から Slack 連携開発を担当しました。

#### プロジェクト規模

- メンバー: 5 ~ 7 人 (自分含む)
- スクラム開発

#### 使用技術

- TypeScript
- React
- Material-UI
- AWS Lambda
- DynamoDB

#### 役割

- フロントエンド開発
- Slack 連携開発

#### プロジェクト詳細

**フロントエンド開発**

業務フローをノードとエッジで視覚化する UI を構築するにあたり、複雑な操作インタラクションの実装が求められました。ReactFlow（現：xyflow）のノードをカスタマイズし、フロー上から直接ノードのデータを編集できる細かな UI / UX を実装することで、画面遷移なしに業務フローを直感的に編集できる体験を実現しました。

**Slack 連携開発**

業務フローにおける進捗変化をリアルタイムで通知する仕組みがなく、担当者間の情報伝達に遅れが生じていました。Slack API の仕様を調査・理解し、Amplify を使用して連携基盤を構築。Slack アカウント連携・Token 管理・各処理時の個人通知機能まで実装しました。退職に際しては全体連携の仕様を文書化してチームに引き継ぎ、後日全機能がリリースされました。

**チーム技術力向上**

参画当初は未経験の技術スタックが多く、学習と開発を並行する必要がありました。チームメンバーのサポートを受けながら急速にキャッチアップし、習熟後は逆にメンバーの技術理解をサポートする立場に転じました。複雑化した処理へのリファクタリングを提案・実施し、輪読会を開催することでチーム全体の技術力向上に貢献しました。
