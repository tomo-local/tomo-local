### 株式会社ハイヤールー (2025/04 ~ 現在)

採用プラットフォームを提供する企業にて、フルスタックエンジニアとして開発に従事しています。採用選考事業と新規事業である母集団形成事業の 2 つのプロジェクトに参画し、フロントエンド開発を中心にバックエンド開発、外部サービス連携まで幅広く担当しています。

#### 母集団形成事業 (2025/08 ~ 現在)

新規事業である母集団形成事業のエンジニアとして参画しています。本事業は、企業の採用ニーズと候補者のスキルを的確にマッチングさせることを目的としており、従来の採用プロセスをより効率的かつ効果的にすることを目指しています。

エージェントチームのメンバーとして、技術的な視点からビジネスに貢献する役割を担っています。企業訪問による採用要件のヒアリングから、候補者のスキルを可視化するシステムの設計・開発まで、エンジニアリングとビジネスの両面で活動しています。特に、候補者の技術力や経験を適切に評価し、企業に分かりやすく伝えるためのレポート機能の改善に注力しており、エージェントチームと連携しながら、より質の高いマッチングを実現するための仕組みづくりに専念しています。

技術面では、フロントエンド開発を中心としながらも、バックエンドの実装も担当し、ユーザー体験を重視した機能開発を推進しています。また、コンペティション機能のリニューアルなど、候補者のスキル評価方法の改善にも積極的に取り組んでいます。

##### プロジェクト詳細

**候補者のスキルレポート UI 改善**

候補者のスキルを可視化するレポートの UI とデザインを改善し、より伝わりやすい表現に刷新しました。エージェントや企業の担当者が候補者のスキルを正確に把握できるようにしました。エージェントチームメンバーとして候補者の職務履歴の深掘りの方法をチームに共有し、レポートの質の向上に貢献しました。

**企業訪問と採用要件ヒアリング**

エンジニアとして企業を訪問し、採用要件のヒアリングを実施しました。技術的な観点から要件を整理し、システムに反映させるための橋渡し役を担いました。
候補者を紹介を行う際に、ヒアリングを行った内容をもとに、企業に対して候補者のスキルがマッチしているかを判定してエージェントに共有しています。

**コンペの問題選択機能実装**

候補者が参加するコンペにて、複数の問題を出題して選択できる機能を実装しました。候補者のスキルレベルに応じた柔軟な評価を可能にしました。

**コンペのリニューアル**

コンペの内容をリニューアルする際に、画面デザインの一新とバックエンド側の処理を実装しました。
フロントエンド開発をメインで担当し、新しい UI の Mock を作成しデザイナーに依頼を行い開発を進めました。

#### 採用選考事業 (2025/04 ~ 2025/08)

採用選考事業において、コーディングテストプラットフォームの機能改善と新機能開発を担当しました。本プラットフォームは、企業がエンジニア候補者の技術力を評価するためのオンラインコーディングテスト・面接を提供するサービスです。

プロジェクトでは、既存機能の改善に加え、ビデオ会議連携やコードレビュー機能といった新規機能の開発を担当しました。特に、Google Meet や Zoom との連携機能では、Google Calendar API を活用した自動 Bot 招待機能を実装し、面接プロセスの自動化を実現しました。また、Monaco Editor を活用したコードレビュー機能では、候補者が提出したコードを視覚的に分かりやすく表示し、レビュアーが効率的に評価できる UI を構築しました。

開発においては、フロントエンドを中心にバックエンド、外部サービス連携まで担当し、ユーザー体験を重視した機能開発に注力しました。処理の最適化やパフォーマンス改善にも取り組み、スムーズな面接体験の提供を実現しています。

##### プロジェクト詳細

**タグ削除機能の追加**

テストやインタビューに付与するタグの削除機能を実装しました。ユーザーが柔軟にタグ管理できるようにすることで、選考管理の利便性を向上させました。

**インタビューのレポート共有機能の実装**

インタビューのレポートを外部に共有できる機能を実装しました。共有する内容の設定、有効期限の設定、共有用 URL の生成機能を開発し、セキュアな情報共有を実現しました。

**Google Meet と Zoom の Bot 招待に関連した Google カレンダー連携**

Google Calendar API との連携をメインで担当し、カレンダー情報の取得と Google OAuth 認証の実装を行いました。Skribby を使用した Meet と Zoom への Bot 招待機能について、スケジュールベースから即時招待へと変更し、バックエンドの処理を最適化しました。これにより、ユーザーがより柔軟に面接を実施できる環境を構築しました。

**新実践形式の UI と提出コードの UI 開発**

Monaco Editor を活用し、提出されたコードを画面上で表示する処理を実装しました。また、初期コードと提出コードの差分表示機能を実装し、レビュアーがコードの変更点を一目で把握できる UI を提供しました。

#### 事業横断の技術改善

事業開発と並行して、プラットフォーム全体の技術的な改善にも取り組みました。

**Query の最適化**

API レスポンスが約 28 秒かかるパフォーマンス問題を調査しました。Cloud SQL のクエリインサイトとアプリケーション側のレイテンシ計測からボトルネックとなるクエリを特定し、EXPLAIN ANALYZE による実行計画の分析を実施。再帰 CTE 内で全カラムを取得していたことが、プランナーの非効率な実行計画選択と temp I/O の発生を招いていることを突き止めました。SELECT 対象を必要最小限のカラムに絞り、JOIN を明示化することで、実行時間を 20.6 倍高速化（3,053ms → 148ms）しました。

**マイクロサービスの統合**

分散していたマイクロサービスの統合を実施し、サービス間通信のオーバーヘッド削減と運用コストの低減に貢献しました。
