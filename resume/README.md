# 職務経歴書

## 職務要約

ERP システムの導入・カスタマイズ開発から採用プラットフォームのフルスタック開発まで、約 5 年にわたり幅広い領域で経験を積んできたエンジニアです。現在は、採用プラットフォーム企業にて、フロントエンド開発を中心にバックエンド開発・外部サービス連携・パフォーマンス改善まで横断的に担当しています。

技術面では、TypeScript/React によるフロントエンド開発と Go によるバックエンド開発を軸に、Kubernetes・Terraform・Google Cloud を活用したインフラ構築まで一貫して対応できます。Monaco Editor を活用したコードレビュー機能や Google Calendar API との連携による面接自動化機能など、ユーザー体験を起点とした機能設計・実装を得意としています。また、EXPLAIN ANALYZE を用いたクエリ最適化（20.6 倍高速化）やマイクロサービスの統合など、プロダクト全体の技術的負債の解消にも主体的に取り組んでいます。

ERP 導入プロジェクトでは、顧客の業務現場でのヒアリングから要件定義・設計・開発・プロジェクト管理まで一貫して担当した経験があり、ビジネス要件を技術に落とし込む力を培いました。現職でもエージェントチームの一員として企業を訪問するなど、技術とビジネスの両面からプロダクトに貢献しています。

業務外では、React Tokyo コミュニティの運営メンバーとして Discord Bot の開発・改善をリードし、スレッド通知機能の実装やコードベースのリファクタリング・自動テスト基盤を構築しました。業務・コミュニティ双方を通じて、チーム全体のコード品質と開発効率の向上に継続的に取り組んでいます。

**タグ削除機能の追加**

**インタビューのレポート共有機能の実装**

インタビューのレポートを外部に共有できる機能を実装しました。共有する内容の設定、有効期限の設定、共有用 URL の生成機能を開発し、セキュアな情報共有を実現しました。

**Google Meet と Zoom の Bot 招待に関連した Google カレンダー連携**

Google Calendar API との連携をメインで担当し、カレンダー情報の取得と Google OAuth 認証を実装しました。Skribby を使用した Meet と Zoom への Bot 招待機能について、スケジュールベースから即時招待へと変更し、バックエンドの処理を最適化しました。これにより、ユーザーがより柔軟に面接を実施できる環境を構築しました。

**新実践形式の UI と提出コードの UI 開発**

Monaco Editor を活用し、提出されたコードを画面上で表示する処理を実装しました。また、初期コードと提出コードの差分表示機能を実装し、レビュアーがコードの変更点を一目で把握できる UI を提供しました。

#### 事業横断の技術改善

事業開発と並行して、プラットフォーム全体の技術的な改善にも取り組みました。

**Query の最適化**

API レスポンスが約 28 秒かかるパフォーマンス問題を調査しました。Cloud SQL のクエリインサイトとアプリケーション側のレイテンシ計測からボトルネックとなるクエリを特定し、EXPLAIN ANALYZE による実行計画の分析を実施。再帰 CTE 内で全カラムを取得していたことが、プランナーの非効率な実行計画選択と temp I/O の発生を招いていることを突き止めました。SELECT 対象を必要最小限のカラムに絞り、JOIN を明示化することで、実行時間を 20.6 倍高速化（3,053ms → 148ms）しました。

**マイクロサービスの統合**

分散していたマイクロサービスの統合を実施し、サービス間通信のオーバーヘッド削減と運用コストの低減に貢献しました。

## その他の活動

### React Tokyo コミュニティ Discord Bot 開発

日本最大級の React コミュニティ「React Tokyo」の運営メンバーとして、コミュニティの活性化を支える Discord Bot の開発・改善をリードしています。

#### 使用技術

- Deno
- TypeScript
- Discord API

#### 開発内容

**スレッド通知機能の設計・実装**

コミュニティ内で「スレッドの議論が埋もれて参加しづらい」という課題に対し、1 時間ごとにアクティブなスレッドを自動検出し、ジェネラルチャンネルへ通知する機能を設計・実装しました。同一スレッドに対する連続通知を防止する重複排除ロジックを実装し、通知の煩雑さを抑えました。また、通知メッセージを Discord Embeds に移行することで、スレッドの内容が視認しやすくなり、ユーザー体験の向上を図りました。

**コードベースのリファクタリングと自動テスト基盤の構築**

Bot の機能追加に伴いコードの複雑性が増していたため、次の改善を提案・実施しました。

- コマンドハンドラのモジュール分割による責務の明確化
- 共通処理の抽出と型安全性の強化
- Deno の標準テストランナーを活用した自動テストの導入

これにより、新規コントリビューターがコードを理解しやすくなり、機能追加時のレビュー工数の削減にもつながりました。

コミュニティの課題を技術で解決する経験を通じて、ユーザー視点での機能設計とチーム開発におけるコード品質の重要性を実感しました。

## 自己 PR

ERP システムのカスタマイズ開発から採用プラットフォームのフルスタック開発まで、約 5 年にわたり多様な領域で経験を積んできたエンジニアです。

ERP 導入プロジェクトでは、お客様の業務現場に赴いてヒアリングを行い、要件定義から設計・開発・プロジェクト管理まで一貫して担当しました。現職でも、エージェントチームの一員として企業訪問による採用要件のヒアリングを実施し、技術的な観点からビジネスに貢献しています。技術とビジネスの両面を理解した上でシステムに落とし込む力は、これまでのキャリアを通じて培ってきた強みです。

### コード品質とチームへの貢献意識

業務委託時代から、既存コードのリファクタリング提案や輪読会の開催など、チーム全体の技術力底上げに取り組んできました。React Tokyo コミュニティでは、Discord Bot のリファクタリングと自動テスト基盤の構築をリードし、新規コントリビューターが参加しやすい環境を整備しました。個人の成果だけでなく、チームとして持続的に開発できる仕組みづくりを大切にしています。

ユーザーの負担を軽減し、業務改善に直結する開発を目指しています。技術力と現場での実践経験を活かし、ユーザーファーストの視点で価値あるプロダクトを届けたいと考えています。
